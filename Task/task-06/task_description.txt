task_description: |
  task-06

  Title:
  Implement Redis Caching Layer for API Performance

  Problem:
  Currently all API routes directly hit MongoDB which creates:
  - High DB load
  - Slow response times
  - No hot-cache for frequently accessed data

  Routes impacted:
  - /api/fetchtextcloud
  - /api/fetchemail
  - /api/fetchimg

  Requirements:

  1. Add Redis as a caching layer.
     - On first request → fetch from MongoDB → cache in Redis
     - On subsequent requests → return cached data

  2. Cache Keys:
     - fetchtextcloud → text:<email>
     - fetchemail → email:<email>
     - fetchimg → img:<email>

  3. Cache TTL:
     - 60 seconds (1 minute)

  4. Cache Invalidation:
     After any of these routes:
     - /api/textcloud
     - /api/deletetextcloud
     - /api/addimage
     - /api/deleteimage

     The relevant cache MUST be cleared.

  5. Redis must be initialized using:
     - utils/redisClient.js

  6. Docker must include Redis container.

  Modified files (minimum):
  - app/api/fetchtextcloud/route.js
  - app/api/fetchemail/route.js
  - app/api/fetchimg/route.js
  - app/api/textcloud/route.js
  - app/api/deletetextcloud/route.js
  - app/api/addimage/route.js
  - app/api/deleteimage/route.js
  - utils/redisClient.js
  - docker-compose.yaml

  Success Criteria:
  ✅ Cached responses returned
  ✅ TTL works
  ✅ Invalidation works
  ✅ No MongoDB query on cache hit
  ✅ Tests must verify Redis usage
